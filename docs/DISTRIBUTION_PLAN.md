# 分发规划

## 推荐分发结构

```text
Chrome Web Store / Edge Add-ons
└─ 浏览器扩展：UI、麦克风采集、草稿、模板、复制

GitHub Releases / 官网下载页
└─ Windows 本地服务：FastAPI、FunASR、模型下载/缓存、启动脚本
```

## 为什么不把模型打进插件包

语音模型体积较大，插件市场审核和更新都不适合承载完整 Python 环境与模型文件。扩展应保持轻量，本地服务和模型由独立安装包负责。

## 内测阶段建议

1. 先通过 GitHub Release 发布本地服务压缩包；
2. 插件使用“开发者模式加载已解压扩展”分发给小范围用户；
3. 收集医院电脑环境：Windows 版本、CPU、内存、是否有显卡、浏览器版本；
4. 记录模型首次下载耗时、冷启动耗时、识别延迟；
5. 稳定后再打包上传插件市场。

## 正式阶段建议

- 插件市场：分发扩展；
- 官网/GitHub：分发本地服务安装器；
- 隐私政策：官网公开 URL；
- 帮助中心：安装、卸载、离线模型、常见问题；
- 授权服务：仅处理授权状态，不上传病历内容。

## 收费版建议

免费版保留可用闭环：听写、编辑、复制、基础模板、热词。Pro 版重点放在效率与管理能力：高级模板、质控规则包、批量导入导出、机构热词包、离线授权、部署管理。

## 多语言说明

插件已支持中文/英文界面切换，并提供中文普通话与英文听写选择。中文听写为当前主要能力，支持流式输入；英文听写为 Beta 能力，依赖本地英文 ASR 模型，当前采用停止录音后批量识别。插件 manifest 已使用 `_locales` 提供中文与英文商店基础信息。
